79 // Parameters:
80 // a) data:
81 // - it contains the RLC layer messages (RLC-PDU) to be transmitted, or the RLC layer messages
82 // that have been received by the MAC sub-layer.
83 //
84 // b) no_tb :
85 // - indicates the number of transport blocks transmitted by the peer entity within the
86 // transmission time interval, based on the TFI value.
87 //
88 // c) buffer_occupancy :
89 // - the parameter Buffer Occupancy (BO) indicates for each logical channel the amount of data
90 // in number of bytes that is available for transmission and retransmission in RLC layer.
91 // When MAC is connected to an AM RLC entity, control PDUs to be transmitted and RLC PDUs
92 // outside the RLC Tx window shall also be included in the BO. RLC PDUs that have been
93 // transmitted but not negatively acknowledged by the peer entity shall not be included in the BO.
94 //
95 // d) rx_timing_deviation, TDD only:
96 // - it contains the RX Timing Deviation as measured by the physical layer for the physical
97 // resources carrying the data of the Message Unit. This parameter is optional and only for
98 // Indication. It is needed for the transfer of the RX Timing Deviation measurement of RACH
99 // transmissions carrying CCCH data to RRC.
100 //
101 // e) no_pdu:
102 // - specifies the number of PDUs that the RLC is permitted to transfer to MAC within a
103 // transmission time interval.
104 //
105 // f) pdu_size:
106 // - specifies the size of PDU that can be transferred to MAC within a transmission time interval.
107 //
108 // g) UE-ID Type Indicator:
109 // - indicates the UE-ID type to be included in MAC for a DCCH when it is mapped onto a common
110 // transport channel (i.e. FACH, RACH, DSCH in FDD or CPCH). On the UE side UE-ID Type Indicator
111 // shall always be set to C-RNTI.
112 //
113 // h) tx_status:
114 // - when set to value "transmission unsuccessful" this parameter indicates to RLC that transmission
115 // of an RLC PDU failed in the previous Transmission Time Interval, when set to value
116 // "transmission successful" this parameter indicates to RLC that the requested RLC PDU(s)
117 // has been submitted for transmission by the physical layer.
118 //
119 // i) rlc_info
120 // - indicates to MAC the configuration parameters that are critical to TFC selection depending
121 // on its mode and the amount of data that could be transmitted at the next TTI. This primitive
122 // is meant to insure that MAC can perform TFC selection.
123 struct rlc_entity_info {
124  unsigned char rlc_protocol_state;
125 };
